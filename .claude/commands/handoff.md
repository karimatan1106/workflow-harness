以下の手順でハンドオフを作成してください：

## ステップ1: 既存 HANDOFF.toon のアーカイブ

`.agent/handoff/HANDOFF.toon` が存在する場合：
- そのファイルの内容の先頭行から日時を取得
- `.agent/handoff/YYYY-MM-DD-HHMM.toon` にリネーム（ローカル時刻・24時間表記）

## ステップ2: CRITICAL.md の鮮度更新

`.agent/CRITICAL.md` を読み込み、今回のセッションで新たに発見したピットフォールがあれば追記してください。
- 新しいバリデーター回避パターン
- 新しい TOON フォーマットの注意点
- 新しいフェーズ固有の落とし穴
- 追記後も全体20行以内を維持すること（古い項目で他でカバーされているものは統合・削除可）

追記がなければそのままにしてください。

## ステップ3: HANDOFF.toon の新規作成

新しい `.agent/handoff/HANDOFF.toon` を以下のテンプレートに従って作成してください。
各項目には、現在までのチャット履歴や作業内容からAI自身が自己の行動を要約し、
具体的な内容を記入してから保存してください。
**単なる空のテンプレートのまま保存してはいけません。**

```
## HANDOFF
date: {YYYY-MM-DD HH:MM}

## 使用ツール
tool: Claude Code (モデル名)

## 現在のタスクと進捗
- [ ] タスク名：現在の状況
（harness_status で取得できるアクティブタスクも含める）

## 試したこと・結果
- 成功したアプローチ（コミットハッシュがあれば記載）
- 失敗したアプローチ（理由と回避策）

## ハーネス運用で学んだこと
- バリデーター回避パターン
- TOON フォーマットの注意点
- フェーズ固有の落とし穴

## 次のセッションで最初にやること
- 最初のアクション
- 次のアクション

## 注意点・ブロッカー
- 注意すべき事項
- 未解決のバグ（VDB-N 等）
```

## ステップ4: HANDOFF.toon の品質セルフチェック

作成した HANDOFF.toon が以下を全て満たすか確認してください：
- [ ] 全6セクション（使用ツール/タスクと進捗/試したこと/学んだこと/次にやること/注意点）が存在する
- [ ] 各セクションに最低1行の具体的内容がある（テンプレートの例文そのままは不可）
- [ ] 「次のセッションで最初にやること」に実行可能なアクションが記載されている
- [ ] コミットハッシュやタスクIDなど、検証可能な参照が含まれている
- 不合格の場合は修正してから保存すること

## ステップ5: 完了報告

「HANDOFF.toon を作成しました」と報告し、以下を表示：
- CRITICAL.md への追記内容（あれば）
- HANDOFF.toon の内容サマリー
- 品質チェック結果
