# regression_test フェーズ

## 概要

リグレッションテストフェーズでは、既存機能の回帰テストを実行し、今回の変更が既存機能に影響を与えていないことを確認します。

## 目的

- 既存機能の動作保証
- 意図しない副作用の早期検出
- 品質の継続的な維持

## 前提条件

- `testing` フェーズが完了していること
- リグレッションテストが存在すること（存在しない場合はスキップ可能）

## 実行内容

### 1. リグレッションテストの実行

#### バックエンド

```bash
cd src/backend && vitest run tests/regression/
```

#### フロントエンド

```bash
cd src/frontend && npm run test -- test/regression/
```

### 2. テスト結果の確認

- 全てのテストがパスすること
- 失敗したテストがある場合は原因を特定

### 3. 失敗時の対応

失敗したテストがある場合:

1. **今回の変更が原因の場合**: 実装を修正し、再度テストを実行
2. **テストの問題の場合**: テストを修正し、再度テストを実行
3. **既知の問題の場合**: 問題を記録し、対応計画を立てる

## 成果物

- テスト実行結果（コンソール出力）
- 必要に応じて `{docsDir}/regression-test-result.md`

## 完了条件

- [ ] 全てのリグレッションテストがパスする
- [ ] 失敗したテストがある場合は対応が完了している

## 次のフェーズ

`parallel_verification` - 並列検証フェーズへ進みます。

## ディレクトリ構成

```
src/backend/tests/regression/
├── README.md
├── setup.ts
└── {task-name}/
    └── *.test.ts

src/frontend/test/regression/
├── README.md
└── {task-name}/
    └── *.test.tsx
```

## 注意事項

- リグレッションテストが存在しない場合でもフェーズは実行される
- 「今回のタスクとは関係ない」という理由でテスト失敗を無視することは禁止
- 失敗したテストは必ず対応する
