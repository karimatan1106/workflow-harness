## 変更内容の確認

### 修正対象ファイル

#### 1. `workflow-plugin/mcp-server/src/state/lock-utils.ts`

**追加されたリトライロジック**:

- `atomicWriteJson`関数内に新規リトライ機構を追加
- `EPERM`（Permission denied）と`EBUSY`（Resource busy）エラーに対応
- 最大3回のリトライを実装（各リトライ間に10ms の sleep を挿入）
- 最終的にリトライが失敗した場合は元のエラーをスロー

この変更は既存の関数シグネチャを変更せず、内部動作のみを改善するため、呼び出し元コードへの影響はありません。

#### 2. `workflow-plugin/mcp-server/src/state/__tests__/lock-utils.test.ts`

**新規作成されたテストファイル**:

- TC-01: 正常系（リトライなしでファイル書き込み成功）を検証
- TC-02: EPERM エラー 1回目で発生→リトライで成功を検証
- TC-03: EBUSY エラー 1回目で発生→リトライで成功を検証
- TC-04: EPERM エラーが3回連続発生→リトライ後もエラーをスロー検証
- TC-05: 他種のエラー（例: EACCES）はリトライせず即座にスロー検証
