## 変更内容の詳細

### ドキュメント構成の強化

**更新前:** インターフェース定義が簡潔で、実装の詳細が不明確

```markdown
### atomicWriteJson
function atomicWriteJson<T>(filePath: string, data: T): void
```

**更新後:** リトライロジックを含めた詳細な説明に拡張

```markdown
### atomicWriteJson（FR-19: リトライロジック追加）

**機能:**
- 一時ファイルにデータを書き込み
- fs.renameSync()で原子的にリネーム
- EPERM / EBUSY エラー発生時は最大3回リトライ
- リトライ前に 100ms の同期スリープ（sleepSync）で待機
- その他のエラーは即座にスロー

**リトライロジック:**
- リトライ対象エラー: EPERM, EBUSY
- リトライ上限: 3回（attempt = 0,1,2,3）
- リトライ間隔: 100ms
- それ以外のエラー: 即座にスロー
```

### 実装の背景と理由

**FR-19 で EPERM/EBUSY リトライを追加した理由:**

Windows/macOS の antivirus ソフトウェアがファイル操作をインターセプトする際、fs.renameSync() が一時的に EPERM または EBUSY エラーを返すことがある。
これにより状態ファイル更新が失敗する問題が発生したため、リトライメカニズムを実装して堅牢性を向上させた。

**sleepSync をローカル定義化した理由:**

- lock-utils.ts 内でのみ使用される内部ヘルパー関数
- SharedArrayBuffer の可用性確認が lock-utils.ts の責任範囲内
- モジュール間の依存性を最小化し、テスト時のモック容易性を向上
