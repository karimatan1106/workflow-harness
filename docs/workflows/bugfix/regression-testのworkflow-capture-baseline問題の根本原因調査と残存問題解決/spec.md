## FR-16 の記録（将来タスク候補）

今回スコープ外として判断した FR-16 の概要を記録する。

`next.ts` には `forceTransition` パラメータに設計上の不整合が存在する。regression_test フェーズへの入場時は `forceTransition=true` でベースライン存在チェックをスキップできるが、regression_test から parallel_verification への遷移時はこのフラグに関係なくベースラインが必須とされている。この非対称な設計により「regression_test に入れたが parallel_verification に進めない」状態が発生しうる。

将来タスクで対処する場合の2方向として、方向Aは regression_test から parallel_verification への遷移時にも forceTransition を有効化して一貫性を持たせる案である。方向Bは regression_test 入場時の forceTransition によるベースライン存在チェックスキップを廃止し、常にベースラインが必要な設計に統一する案である。どちらの方向が適切かは別タスクの research フェーズで判断するものとする。
方向Aと方向Bはいずれも設計上の一貫性を目標とするが、影響範囲と後方互換性が異なるため、研究フェーズでの詳細分析が必要である。
FR-16 の優先度は低から中程度と評価されており、実際の問題が報告された場合に対処すれば十分と判断されている。
