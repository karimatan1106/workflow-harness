## サマリー

本ドキュメントは、RCA-1タスク（next.tsのデッドコード削除）の実行中に発生した残存問題を解決するための要件定義を記述する。

調査フェーズで明らかになった主要事項:
- `workflow-plugin/mcp-server/src/verify-sync.test.ts`（以下「誤配置ファイル」）がgit管理外のまま存在している。ただしgitはこのファイルを未追跡と扱っており、`git status` の出力に表示されない状態になっている（サブモジュール内のgitignoreまたはgit内部処理による可能性がある）
- 正規のテストファイルは `mcp-server/src/__tests__/verify-sync.test.ts`（以下「正規ファイル」）であり、gitに追跡済みでかつ内容も最新の正しい状態である
- テストカウントの増加（852件から897件へ）は、バグ修正に伴う意図的なテスト追加によるものであり、リグレッション問題は存在しない
- コミット5d0450fによるnext.tsのデッドコード削除は正しく適用済みであり、追加対応は不要である

次フェーズ（spec.md作成）で必要な情報:
- 誤配置ファイルの削除は implementation フェーズで `rm` コマンドを使用して実施する
- 正規ファイルへの変更は不要であることが確認された
- テストスイート実行による回帰確認は testing フェーズで実施する

## 機能要件

### FR-1: 誤配置テストファイルの削除

誤配置ファイル `workflow-plugin/mcp-server/src/verify-sync.test.ts` を削除しなければならない。

このファイルは以下の理由で削除が必要である:
- RCA-1タスクのsubagentが誤ったパスにテストファイルを作成したことで生じた産物であり、意図して作成されたものではない
- 内容が正規ファイルより古い版（vi.mockのパスが誤っており、25フェーズ対応が未反映）であるため、現在の仕様に適合しない
- gitに追跡されていないにもかかわらずディスク上に存在することで、ディレクトリ構造が混乱し、今後の開発者が正規ファイルの場所を誤認するリスクがある

削除対象ファイルの識別情報:
- パス: `workflow-plugin/mcp-server/src/verify-sync.test.ts`（サブモジュール内の相対パスは `mcp-server/src/verify-sync.test.ts`）
- ファイルサイズ: 826行、38,657バイト
- 作成日時: 2026年2月19日 20:49（正規ファイルより4分早いタイムスタンプ）
- gitの追跡状態: 未追跡（git ls-filesに表示されない）

### FR-2: 正規テストファイルへの変更禁止

正規ファイル `mcp-server/src/__tests__/verify-sync.test.ts` は変更してはならない。

このファイルは以下の状態であることが確認されており、変更を加える理由が存在しない:
- vi.mockのパスが `../phases/definitions.js` と正しく設定されており、`__tests__` ディレクトリからの相対パスとして適切である
- 25フェーズ（非並列フェーズ14件 + サブフェーズ11件）に対応した期待値が設定されており、現行の仕様に合致している
- gitに追跡済みであり、コミットb642cc2で正式に追加されたファイルである

### FR-3: テストスイートの正常動作確認

誤配置ファイルを削除した後、テストスイート全体が引き続きパスすることを確認しなければならない。

確認すべき事項は以下の通りである:
- 削除により既存のテスト実行に影響が生じていないこと
- 正規ファイル（`__tests__/verify-sync.test.ts`）のすべてのテストケースがパスすること
- テスト総数が削除前後で変わらないこと（誤配置ファイルはgit未追跡であり、テストランナーによっては実行されている可能性があるため、テスト数の変動を記録する）

## 非機能要件

### NFR-1: 変更の最小性

本タスクで行う変更は、誤配置ファイルの削除のみに限定しなければならない。

変更スコープを最小に保つ理由:
- 誤配置ファイル以外の問題はすべてRCA-1タスクで解決済みである
- 余分な変更は意図しない副作用を引き起こすリスクを高める
- コードレビューの負荷を最小化するために、単一の明確な目的を持つコミットとして記録することが望ましい

具体的な制約:
- `next.ts` への追加変更は行わない（コミット5d0450fで対応完了済み）
- `__tests__/verify-sync.test.ts` への変更は行わない（現在の状態が正しい）
- その他のソースファイルへの変更は本タスクのスコープ外である

### NFR-2: Gitリポジトリの整合性維持

削除操作はgitの追跡状態に影響を与えてはならない。

具体的な制約:
- 誤配置ファイルはgit未追跡であるため、`git rm` ではなく通常の `rm` コマンドで削除する
- 削除後に `git status` でワーキングツリーがクリーンであることを確認する
- 削除操作自体をコミットする必要はないが、他の変更と合わせてコミットする場合は明確なコミットメッセージを付与する

### NFR-3: テスト実行環境の安定性

テストスイートは削除操作の前後で安定した状態を維持しなければならない。

評価基準:
- 誤配置ファイルがテストランナー（vitest）によって実行されていた場合、削除によりテスト総数が減少することがある。この場合は新しいテスト総数を正しいベースラインとして記録する
- 誤配置ファイルがテストランナーから除外されていた場合、削除後もテスト総数は変わらない
- いずれの場合も、テストの失敗件数がゼロであることが要件を満たすための条件である

### NFR-4: 作業の可逆性

削除操作は理論上復元可能な状態で実施することが望ましいが、実用上は復元不要である。

根拠:
- 誤配置ファイルの内容は正規ファイルのサブセット（古い版）であり、正規ファイルが存在する限り情報の損失は発生しない
- gitに未追跡のファイルであるため、削除してもgit履歴には影響しない
- 万が一削除が誤りであると判明した場合は、正規ファイルの内容を参照して再構築できる

## スコープ外の事項

以下は本タスクの要件に含まれない:

テストカウント変動（852件から897件への増加）に関する追加対応は不要である。調査フェーズにおいて、この増加は4ead400コミットによるバグ修正テストの追加（`bug-fix-regression-transition.test.ts` の新規追加）とverify-syncテストの追加（コミットb642cc2）によるものと特定されており、意図的なテストカバレッジ拡充であることが確認された。

next.tsに対する追加のコード修正は不要である。コミット5d0450fで実施されたデッドコード削除（常に偽となる `currentPhase !== 'regression_test'` の条件分岐と未使用の `recordTestOutputHash` importの削除）は正しく適用済みであり、現在のコードは期待通りの状態にある。

メインリポジトリの未コミット変更（削除済みの古いワークフロー状態ファイル群、動画ファイル群）への対応は、本タスクとは独立した問題であるため、本要件定義の対象外とする。
