## テスト方針

本タスクの変更対象は2ファイルのみであり、いずれもテンプレート文字列または自然言語ドキュメントへの追記である。
ロジック変更を含まないため、単体テストの新規作成は行わず、静的検証と既存自動テストによるリグレッション確認を主軸とする。

静的検証の方針として、Readツールを使用して変更後のファイル内容を直接確認し、追加内容が仕様通りであることを目視で検証する。
自動テスト検証の方針として、`workflow-plugin/mcp-server` ディレクトリの既存テストスイートを実行し、追記による既存動作への影響がないことを確認する。

FR-20の検証では、security_scan subagentTemplate文字列内に新しいセクションが正しく挿入されていること、禁止語を含まないこと、5観点の説明が全て含まれることを確認する。
FR-21の検証では、CLAUDE.mdの「AIへの厳命」セクション末尾にルール23が追記されていること、sessionTokenとworkflow_statusへの言及が含まれることを確認する。

実装完了後の必須手順として、npm run buildによるトランスパイル実行とMCPサーバー再起動が義務付けられる。
MCPサーバー再起動後にworkflow_statusを呼び出し、現在のフェーズが正常に返ることを確認することで最終的な動作検証とする。
