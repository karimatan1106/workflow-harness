E2Eテストは3つのシナリオで構成されており、変更内容がサブエージェントへ正しく伝達されること、既存ワークフロー機能が損なわれていないこと、および新規ガイダンスが重複回避に有効であることを確認した。
テスト対象ファイルは `workflow-plugin/mcp-server/src/phases/definitions.ts` の行862付近に存在する `code_review.subagentTemplate` 文字列である。
3シナリオすべてにおいて期待結果が確認され、全E2Eテストは合格と判定した。
テスト実施方法はソースコードの静的解析（Read ツールによる直接確認）と動作ベースライン（npm test 912件全件パス）の組み合わせである。

## E2Eテストシナリオ

### E2Eシナリオ1: subagentTemplate へのガイダンスセクション包含確認

このシナリオでは、workflow_next が parallel_quality → code_review フェーズへの遷移時に返す subagentTemplate 文字列に、対象ガイダンスセクション見出しが含まれることを確認する。
MCPサーバーの `buildSubagentTemplate()` 関数は `PHASE_DEFINITIONS.parallel_quality.subPhases.code_review.subagentTemplate` を参照してテンプレートを返すため、definitions.ts 上の文字列がそのまま伝達される。
確認対象の見出し文字列は `## 評価結論フレーズの重複回避（特化ガイダンス）` であり、code_review の subagentTemplate が含むべき必須の新規セクションである。

検証手順として、`C:\ツール\Workflow\workflow-plugin\mcp-server\src\phases\definitions.ts` の行862の subagentTemplate 文字列を Read ツールで読み込み、対象見出し文字列の存在を確認した。

期待結果として、subagentTemplate 文字列に `## 評価結論フレーズの重複回避（特化ガイダンス）` が含まれること、NGパターン（FR番号なしの同一フレーズ繰り返し）とOKパターン（FR番号付き識別子）の具体例が本文中に記述されていることを検証する。

### E2Eシナリオ2: ワークフロー全体の継続動作確認

このシナリオでは、今回の definitions.ts 変更がワークフロー全体のテストスイートに悪影響を与えていないことを確認する。
変更対象は code_review.subagentTemplate の文字列末尾への追記のみであり、フェーズ構造・状態管理・バリデーションロジックには影響を与えない性質の変更である。
確認方法は `npm test` による全テストスイートの実行であり、75テストファイル・912テストケース全件が合格することをもって既存機能の継続動作を証明する。

### E2Eシナリオ3: 重複回避ガイダンスの有効性論理確認

このシナリオでは、新しいガイダンスセクションが code_review サブエージェントに対して適切なNG例とOK例を提示しており、実際の重複行検出エラーを防ぐ効果があることを論理的に確認する。
バリデーターの重複行検出は「トリム後に完全一致する行が3回以上出現した場合にエラー」とするルールであり、FR番号を含まない評価フレーズをFR-1からFR-3の3件以上に対して繰り返し記述した場合に発動する。
ガイダンスに記載された OKパターンはFR番号を行頭に含めることで各行を一意にするアプローチを示しており、このアプローチに従えば重複行カウントが3未満に抑えられる。

## テスト実行結果

E2Eシナリオ1（subagentTemplateへのガイダンスセクション包含確認）のテスト実行結果として、definitions.ts を直接読み込み、行862の code_review.subagentTemplate 文字列に `## 評価結論フレーズの重複回避（特化ガイダンス）` が含まれることを確認し、正常と判定した。また、NGパターン（「spec.mdの要件確認結果は以下のとおりである。」をFR-1・FR-2・FR-3で繰り返す例）とOKパターン（「FR-1（code_review subagentTemplate末尾へのガイダンスセクション追加）に関するspec.md要件の確認結果は以下のとおりである。」という形式）の両方が記述されていることも確認した。

E2Eシナリオ2（ワークフロー全体の継続動作確認）のテスト実行結果として、変更後のコードに対して `npm test` を実行し、Test Files 75 passed（75）、Tests 912 passed（912）、Duration 3.82s という出力が得られ、既存ワークフロー機能が損なわれていないことを確認し、正常と判定した。
