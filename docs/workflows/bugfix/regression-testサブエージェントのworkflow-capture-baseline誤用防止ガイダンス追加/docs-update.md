# ドキュメント更新レポート: regression_test サブエージェントの workflow_capture_baseline 誤用防止ガイダンス追加

## サマリー

- 目的: 本タスクで実装された FR-13 と FR-14 の変更が、永続的なドキュメント（`docs/spec/`、`docs/architecture/` など）への更新を要求するかどうかを評価する
- 評価スコープ: `docs/spec/`（機能仕様書）、`docs/architecture/`（システム設計書）、`CLAUDE.md`（プロジェクトガイドライン）、その他の永続ドキュメント
- 主要な決定事項: 本タスクの変更は MCP プラグイン内部の subagentTemplate 文字列修正に限定されており、ユーザー向けの機能・API・画面設計に変更がないため、永続ドキュメントの更新は不要と判定した
- 評価結果: 永続ドキュメント更新対象なし
- 次フェーズで必要な情報: testing フェーズで `npm test` を実行し全テストケースの合格を確認、その後 `npm run build` でトランスパイルして MCP サーバーを再起動すること

## 更新対象の検討

### 1. docs/spec/ 配下のドキュメント（機能仕様書）

spec.md に記載されたとおり、本タスクの変更は `workflow-plugin/mcp-server/src/phases/definitions.ts` の `regression_test.subagentTemplate` フィールドへの文字列追記のみである。

本変更は以下の理由により `docs/spec/` への反映が不要である:

- **ユーザー向けの機能変更がない**: FR-13 と FR-14 は subagent が従うべき指示を明確化するための内部ガイダンス強化であり、API エンドポイント・UI 機能・ビジネスロジックには変更がない
- **ワークフロー機能の仕様変更がない**: regression_test フェーズが実施する処理内容や条件は不変であり、機能仕様書（`docs/spec/features/regression-testing.md` など）に記載された要件に変更はない
- **ユーザーが認識すべき仕様変更がない**: 本変更は subagent の内部動作を改善するものであり、エンドユーザーやシステム利用者に通知すべき仕様追加・変更ではない

### 2. docs/architecture/ 配下のドキュメント（システム設計書）

本タスク前後でシステムアーキテクチャ・フェーズ構成・層間の依存関係に変化はない。

以下の理由により `docs/architecture/` への反映が不要である:

- **アーキテクチャの変更がない**: MCP サーバー内部の制御フローやサブエージェント間の通信プロトコルに変更がない。FR-13・FR-14 はサブエージェントへの指示内容の強化であり、システム構成図・決定記録（ADR）等を更新する必要がない
- **层间の依存関係変化がない**: regression_test フェーズが依存する MCP ツール（`workflow_get_test_info`・`workflow_record_test_result` など）の定義・実装に変更がない
- **外部インターフェース変化がない**: API 仕様書（`docs/architecture/integrations/` など）への反映が不要である

### 3. docs/architecture/decisions/ 配下のドキュメント（ADR）

新たな設計判断が記録される必要があるかを検討する。

本タスクの変更は既存のアーキテクチャ決定（FR-4・FR-5 で実装された「subagent のワークフロー制御ツール禁止」）の**強化・補完**であり、新たな設計判断の開始ではない。FR-13・FR-14 は以下の既存決定に基づいている:

- FR-4（testing フェーズの禁止指示）: testing フェーズのサブエージェントはワークフロー制御ツールを禁止される
- FR-5（regression_test フェーズの禁止指示）: regression_test フェーズのサブエージェントはワークフロー制御ツールを禁止される

FR-13・FR-14 は、この既存決定が一部不完全（workflow_capture_baseline が禁止リストに漏れていた）であったことを修正するものであり、新たな ADR を記録する必要はない。

### 4. CLAUDE.md への反映

CLAUDE.md は本プロジェクトの開発ガイドライン・ワークフロープラグイン使用時の必須ルールを記載するドキュメントである。

CLAUDE.md に本タスクの変更を反映する必要があるかを検討する結果、反映は不要と判定した。理由は以下の通りである:

- **既に記載済みの内容**: CLAUDE.md には「ワークフロー制御の4フェーズ（requirements・design_review・test_design・code_review）では workflow_approve を呼び出すこと」および「regression_test フェーズではワークフロー制御ツールの呼び出しを禁止」という指示が既に記載されている（AIへの厳命 7・21）
- **本タスク変更の性質**: FR-13・FR-14 は CLAUDE.md に既に記載されたルール（subagent のワークフロー制御禁止）を **MCP プラグイン側で実装・強化する** ものであり、CLAUDE.md のルール文言の追加・修正を要求しない
- **ユーザー向けガイダンスの変化なし**: CLAUDE.md は人間のユーザー（Orchestrator 実装者）向けガイドであり、subagent テンプレートの内容変化は CLAUDE.md のアップデートを必要としない

### 5. docs/testing/ 配下のドキュメント（テスト計画・結果）

本タスク前後でテスト計画や既存テストの仕様に変化はない。TC-FIX シリーズのテストケース追加は以下の理由により `docs/testing/` への記録が不要である:

- **テストコード内に記録済み**: テストケースは `workflow-plugin/mcp-server/src/__tests__/definitions-subagent-template.test.ts` に実装済みであり、テストファイル自体がテスト仕様の最新ソースである
- **ユニットテストの追加**: TC-FIX テストは定義ファイルの単体テストであり、統合テスト計画書・テスト結果レポートの範囲外である
- **テスト設計の遡及**: TC-FIX は本タスク実装時点で新規追加されたテストケースであり、テスト設計フェーズ（test_design）で記録された計画に含まれていない。実装段階での新規テスト追加は自然な進化であり、計画書の遡及修正を要求しない

## 更新結果

### ドキュメント更新の判定: 不要

本タスクで実装された FR-13（禁止ツールリストへの workflow_capture_baseline 追加）と FR-14（ベースライン前提条件セクション追加）は、以下の理由により **永続的なドキュメント（docs/spec/、docs/architecture/、CLAUDE.md など）への更新を要求しない**と判定した。

#### 判定理由の詳細

1. **変更の性質**: 本タスク変更は MCP プラグイン内部の subagentTemplate 文字列への追記であり、システムの対外インターフェース・機能仕様・アーキテクチャには影響を及ぼさない

2. **ユーザーへの影響なし**: FR-13・FR-14 による指示強化は subagent の誤用防止を目的としており、エンドユーザーやシステム利用者の体験・操作方法に変化をもたらさない

3. **既存ガイドラインの実装化**: CLAUDE.md に既に記載されたルール（subagent のワークフロー制御禁止）を、MCP サーバー側が動作レベルで強制・補完する実装であり、新たなルール・方針の追加ではない

4. **テスト追加の自然な進化**: TC-FIX シリーズのテストケース追加は、実装フェーズで新たに発見された検証項目をテストコードに反映したものであり、テスト計画書の遡及修正対象外である

#### 納期・整合性への影響

本判定に基づき、以下が確保されている:

- **ドキュメント整合性**: spec.md・code-review.md で記載された変更範囲（definitions.ts のみ）と実装内容が一致しており、ドキュメント・実装間に齟齬なし
- **スコープ確認**: spec.md の「変更対象ファイル」セクションが definitions.ts に限定していることが、実装確認（code-review.md の「設計-実装整合性」セクション）で検証済み
- **後続フェーズへの伝達**: testing フェーズで `npm test` を実行し全テストケース合格を確認、その後 `npm run build` と MCP サーバー再起動を実施することが CLAUDE.md のルール 22（コアモジュール変更後の再起動要件）に従って必須となることを確認済み

## 最終判定

本タスクの実装内容（FR-13・FR-14 の追加、TC-FIX テストケース 5 件）に対して、永続的なドキュメント（`docs/spec/`・`docs/architecture/`・`docs/testing/`・`CLAUDE.md` など）への更新は **不要**と判定する。

次フェーズ（testing）では以下を実施すること:

- `npm test` でテストスイート全体（既存 + TC-FIX-1 から TC-FIX-2c）の合格を確認
- `npm run build` でトランスパイル
- MCP サーバープロセスを再起動
- `workflow_status` でフェーズ確認
