# Performance Test Report: hearing-worker-real-choices

## summary

hearing-worker.md と defs-stage0.ts への品質ルール追加が、ランタイム性能およびテスト実行速度に与える影響を分析した。変更は静的テキストの追加のみであり、計測可能なパフォーマンス劣化は検出されなかった。

## baseline

- テストスイート(変更前): 838テスト / 6.71s
- テストスイート(変更後): 843テスト / 6.86s
- 差分: +5テスト / +0.15s (+2.2%)

## analysis

### hearing-worker.md (27行 → 35行, +8行)

エージェント定義ファイルはClaude Codeがセッション開始時に1回だけ読み込む静的テキストである。8行(約300バイト)の追加はコンテキストウィンドウ消費量として無視可能な規模であり、推論速度やトークン処理に測定可能な影響を与えない。ランタイム中の再読み込みは発生しない。

### defs-stage0.ts (44行 → 48行, +3行)

Node.jsモジュールロード時に1回評価されるテンプレートリテラル文字列への追加である。3行(約180バイト)の文字列追加はV8エンジンの文字列割り当てにおいて計測不能なレベルの差分であり、ランタイムパフォーマンスへの影響はゼロと判定する。テンプレート文字列はhearingフェーズ呼び出し時にのみ参照され、他フェーズの実行パスに影響しない。

### hearing-template.test.ts (新規5テスト追加)

テスト実行時のみに影響し、本番コードパスには関与しない。5テスト追加による実行時間増加は+0.15sであり、1テストあたり約30msの標準的な実行コストと一致する。テストスイート全体の実行時間は7秒未満を維持しており、CI/CDパイプラインのボトルネックにはならない。

### メモリ影響

変更対象はすべて静的テキスト(エージェント定義、テンプレートリテラル)であり、動的なメモリ確保パターンに変化はない。ヒープ使用量の増加は合計約500バイト未満であり、Node.jsプロセスのメモリ使用量に対して無視可能である。

## decisions

- PT-001: テストスイート全体の実行時間が6.86s(変更前6.71s)であり、+2.2%の増加は新規5テスト追加分として妥当と判定した。パフォーマンス閾値(10%以内)を満たしている。
- PT-002: hearing-worker.mdの8行追加はセッション開始時の1回読み込みのみに影響し、ランタイム中のホットパスに関与しないため、性能劣化なしと判定した。
- PT-003: defs-stage0.tsの3行追加はモジュールロード時の静的文字列評価のみであり、関数呼び出しやI/O操作を含まないため、ランタイムコストはゼロと判定した。
- PT-004: 変更対象3ファイルのうち本番コードパスに含まれるのはdefs-stage0.tsのみであり、影響範囲がhearingフェーズのテンプレート生成に限定されることを確認した。他フェーズへの波及はない。
- PT-005: メモリ影響は静的テキスト追加(合計約500バイト)のみであり、GCパターンやヒープ成長率に変化を与えないと判定した。
