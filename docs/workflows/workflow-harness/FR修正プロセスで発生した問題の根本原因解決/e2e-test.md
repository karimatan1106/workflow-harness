- 目的: FR-A〜FR-Eの5件の修正が `definitions.ts` に正しく適用されているかを静的検証によるE2Eテストで確認する

- 主要な確認事項: requiredSections配列の要素数・内容、buildPrompt関数の禁止語ループ削除、各フェーズのsubagentTemplateへのガイダンス追記

### シナリオ1: FR-A — code_reviewのrequiredSections配列に「パフォーマンス」が追加されていること

**検証対象の行範囲:** 819〜821行目付近の code_review サブフェーズ定義

**テスト手順:**
1. `definitions.ts` の 821行目付近を読み込み、`code_review` の `requiredSections` 配列を抽出する
2. 配列の要素数が5件であることを確認する
3. 末尾要素が `'## パフォーマンス'` であることを確認する
4. 既存の4要素（サマリー・設計-実装整合性・コード品質・セキュリティ）が順序を保って存在することを確認する

**期待する配列の内容:**
`['## サマリー', '## 設計-実装整合性', '## コード品質', '## セキュリティ', '## パフォーマンス']`

**整合性の確認観点:**

**検証対象の行範囲:** 1093〜1098行目付近の buildPrompt 関数内の禁止語セクション生成部分
