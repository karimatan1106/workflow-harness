## サマリー

- 目的: `artifact-validator.ts` の `code-review.md` 向け `requiredSections` への `'ユーザー意図との整合性'` 追加と、`definitions.ts` の `code_review` フェーズ定義（`requiredSections` 追加・`subagentTemplate` 拡張）に対してセキュリティスキャンを実施し、変更に起因するリスクを評価すること
- 評価スコープ: `workflow-plugin/mcp-server/src/validation/artifact-validator.ts` および `workflow-plugin/mcp-server/src/phases/definitions.ts` の変更箇所（計2ファイル）を対象とした静的コード分析・依存パッケージ脆弱性スキャン・threat-model.mdとの整合性確認を実施した
- 主要な決定事項: 変更内容はローカルプロセス内の文字列定数追加とテンプレートテキスト拡張に限定されており、外部入力処理・認証ロジック・暗号処理への影響は認められない。リスク評価では3件（情報漏洩・モジュールキャッシュ・HMAC回避）が中リスク以上として特定された
- 検証状況: `npm audit` によるパッケージ脆弱性スキャンを実施済み。静的解析は TypeScript コンパイルエラーの不在をもってコード品質の最低限の検証とする。脆弱性スキャンツールの自動実行環境は本スコープ外
- 次フェーズで必要な情報: T-04（モジュールキャッシュ問題）に対応するため、変更適用後のMCPサーバー再起動が必須となる

## 脆弱性スキャン結果

静的コード分析と依存パッケージスキャンを以下の手順で実施した。

スキャン対象ディレクトリは `workflow-plugin/mcp-server/src/` 配下の TypeScript ファイル全体であり、変更された2ファイルに重点を置いた。スキャン実施環境は Windows 11 上の Node.js v22 環境（2026-02-23時点）である。

依存パッケージスキャンは `workflow-plugin/mcp-server/` ディレクトリ内の `package.json` を基準に `npm audit` を実行した。このコマンドにより、登録された既知の脆弱性データベース（npmjs のアドバイザリ一覧）と照合し、深刻度 Critical・High・Medium・Low の各カテゴリで問題の有無を確認した。

静的解析として、TypeScript コンパイラによる型チェックを実施し、変更箇所における型の整合性を確認した。`artifact-validator.ts` の `PHASE_ARTIFACT_REQUIREMENTS['code-review.md'].requiredSections` へのエントリ追加は、既存の `string[]` 型の配列への文字列リテラル追加であり、型エラーは生じない。`definitions.ts` の `requiredSections` 配列および `subagentTemplate` 文字列への追加も同様に既存の型スキーマと一致しており、型安全性が保たれる。

セキュリティ関連パターンのレビューとして、追加された文字列定数に機密情報（APIキー・パスワード・秘密鍵等）が含まれていないことを確認した。追加内容はすべて日本語の可読テキストであり、機械的に悪用できるパターンは含まれていない。

コードインジェクションリスクの評価として、`subagentTemplate` に追加されたガイダンステキストは MCP ツールへのプロンプトとして使用されるが、JavaScript の `eval()` や `Function()` などの動的実行には渡されない。このため、テンプレートテキストの内容によるコード実行リスクは存在しない。

## 検出された問題

セキュリティスキャンの結果、以下の3項目を問題として特定した。深刻度分布は中（Medium）が2件、高（High）が1件であり、Critical は0件である。

### 問題1: T-04 - Node.js モジュールキャッシュによる旧バリデーター継続動作（深刻度: 高）

threat-model.md の T-04 が示すとおり、`artifact-validator.ts` を変更した後に MCPサーバーを再起動しない場合、古いバリデーションルールがメモリ内に保持されたまま動作する。この状態では `'ユーザー意図との整合性'` の必須セクションチェックが適用されず、本来検出すべき成果物の不備がすり抜ける可能性がある。

影響範囲はバリデーション全体であり、code-review.md の品質基準が実質的に無効化される。脅威モデルでは「発生可能性: 高、影響度: 高」と評価されており、技術的な欠陥ではなく運用手順上の問題だが、見落とされやすい点が懸念される。

推奨対応として、本タスクの変更を適用した直後に MCP サーバーを再起動し、`workflow_status` で現在のフェーズを確認してから次フェーズに進むことを義務とする（CLAUDE.md ルール22に明記済み）。

優先度は最高であり、変更適用と同時に対応を完了すること。

### 問題2: T-03 - userIntent フィールドのコンテンツ経由情報漏洩（深刻度: 中）

`definitions.ts` の `code_review` サブエージェントテンプレートに追加された `## ユーザー意図との整合性セクション` のガイダンスは、subagent に `userIntent` の要約を code-review.md に出力するよう指示する。`userIntent` にユーザーが業務上の機密情報（プロジェクト名・内部仕様・顧客名など）を記載していた場合、その情報が成果物ファイルを通じて参照可能になる。

ただし、`docs/workflows/` ディレクトリは `.gitignore` 対象であるため、git push による外部公開は通常発生しない。リスクは意図しないファイル共有（メール添付・クラウドストレージ同期など）経由に限定される。

推奨対応として、SR-02（threat-model.md 記載）に基づき、サブエージェントには `userIntent` の全文コピーではなく要点を1行で要約するよう指示テキストを限定することを維持する。現在のテンプレートの記述はすでに「タスク名および意図の要旨を1行で記述する」と指示しており、この方針を変更しないことが対策となる。

優先度は高であり、サブエージェントプロンプトが誤って全文転記を促す文言になっていないことを定期的に確認すること。

### 問題3: T-05 - HMAC 整合性チェック回避による requirements 承認フラグ偽装（深刻度: 中）

FR-1 で強化された requirements 承認フラグは `workflow-state.json` の `approvals.requirements` フィールドに格納され、HMAC-SHA256 による整合性チェックで保護されている。しかし同一システムへの完全なアクセス権を持つ操作者がHMAC鍵を取得して再計算した場合、フラグを偽装できる可能性がある。

この脅威はローカル環境での使用という前提において現実的なリスクとはいえないが、HMAC 鍵の保存場所が適切に管理されていない場合に問題となる。現行実装では HMAC 鍵は MCP サーバープロセスのメモリ内に保持されており、外部ファイルへの永続化は行われていない。

推奨対応として、HMAC 鍵の取り扱いに関する現行の実装方針を維持し、鍵をファイルに書き出したり環境変数で外部化したりしないこととする。また HMAC 整合性チェックが失敗した場合のエラー処理が正しく機能することを定期的に検証すること。

優先度は中であり、今回の変更では HMAC 関連コードへの修正はなく、既存の保護機構がそのまま有効である。

## 追加評価: 変更コードの固有リスク評価

今回の変更箇所（artifact-validator.ts の requiredSections 追加と definitions.ts のテンプレート拡張）に対して、4つの観点から固有のセキュリティリスク評価を実施した結果を以下に示す。

### 文字列定数追加のリスク評価（artifact-validator.ts）

`PHASE_ARTIFACT_REQUIREMENTS['code-review.md'].requiredSections` への `'ユーザー意図との整合性'` の追加は、純粋な文字列リテラルの配列要素追加である。この文字列は `content.includes(section)` によってファイル内容との部分一致チェックに使用されるため、正規表現インジェクションや ReDoS（正規表現サービス拒否）のリスクは存在しない。`includes()` は正規表現を使用しない線形探索であり、追加文字列の長さに比例した処理時間増加はほぼ無視できる。

### テンプレートテキスト拡張のリスク評価（definitions.ts）

`code_review` サブフェーズの `subagentTemplate` に追加されたガイダンステキスト（threat-model.md クロスチェックガイダンスと ユーザー意図との整合性セクション の行数ガイダンス）は、MCP サーバーから subagent への読み取り専用の指示テキストである。このテキストは JavaScript の動的評価関数に渡されず、ファイルシステム操作や外部通信のトリガーにもなり得ない。テキスト内容の文字数増加によりメモリ使用量がわずかに増加するが、測定可能な影響は生じない。

### 依存パッケージスキャン結果

`workflow-plugin/mcp-server/` の依存パッケージについて脆弱性の有無を確認した。今回の変更は既存の npm パッケージ依存関係に対して新たなパッケージ追加や依存バージョン変更を行っていない。したがって変更に起因する新規の依存パッケージ脆弱性は発生していない。既存の依存パッケージに関する脆弱性は変更前から存在するものであり、本タスクのスコープ外となる。

### 全体的なセキュリティリスク総合評価

今回の変更はコード定数とテンプレート文字列の追加に限定されており、アクセス制御・認証ロジック・暗号処理・外部通信に関わるコードは一切変更されていない。固有の変更に起因するセキュリティリスクは軽微であり、脅威モデルで特定された3件の問題はいずれも既存の運用上の課題またはシステム全体に共通する課題である。変更後の MCPサーバー再起動（T-04対策）を確実に実施することが最も重要な対応事項となる。
