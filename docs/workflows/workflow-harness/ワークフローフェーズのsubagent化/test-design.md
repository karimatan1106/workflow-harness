# テスト設計: ワークフローフェーズのsubagent化

## 1. テスト方針

本機能はCLAUDE.mdの設定追加とプロンプトテンプレートの作成であり、コード実装を伴わない。テストは手動検証を中心とする。

## 2. テストケース

### TC-001: CLAUDE.md読み込み確認

**目的**: 追加した設定がClaude Codeに正しく読み込まれることを確認

**手順**:
1. Claude Codeを起動
2. workflow-pluginプロジェクトを開く
3. `/workflow status`を実行

**期待結果**: エラーなく動作する

---

### TC-002: subagent実行ルールの確認

**目的**: CLAUDE.mdに追加したsubagent実行ルールが適用されることを確認

**手順**:
1. 新しいタスクを開始: `/workflow start テストタスク`
2. researchフェーズでsubagent起動の指示が表示されるか確認

**期待結果**:
- subagent_type、model、入出力ファイルの指示が含まれる
- Task toolの使用が指示される

---

### TC-003: プロンプトテンプレートの存在確認

**目的**: 各フェーズのプロンプトテンプレートが存在することを確認

**手順**:
1. `skills/workflow/phases/`ディレクトリを確認
2. 各フェーズのテンプレートファイルが存在するか確認

**期待結果**:
- research.md が存在
- requirements.md が存在
- （以下、全フェーズ分）

---

### TC-004: 並列フェーズ設定の確認

**目的**: parallel_*フェーズで複数subagent起動の指示があることを確認

**手順**:
1. parallel_analysisフェーズまで進める
2. 設定を確認

**期待結果**:
- threat_modelingとplanningの並列起動が指示される
- 両方完了後にworkflow_complete_subの指示がある

---
