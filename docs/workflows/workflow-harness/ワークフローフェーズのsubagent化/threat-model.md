# 脅威モデル: ワークフローフェーズのsubagent化

## 1. システム概要

本機能は、workflow-pluginの各フェーズをClaude CodeのTask tool（subagent）で実行するように変更する。

## 2. 脅威分析

### 2.1 資産

| 資産 | 機密性 | 完全性 | 可用性 |
|------|--------|--------|--------|
| ワークフロー成果物（docs/workflows/） | 低 | 高 | 中 |
| ワークフロー状態（.claude/state/） | 低 | 高 | 高 |
| ソースコード | 中 | 高 | 高 |

### 2.2 脅威と対策

| 脅威 | リスク | 対策 |
|------|--------|------|
| subagentによる意図しないファイル変更 | 中 | フェーズごとの編集可能ファイル制限（既存） |
| subagent間の成果物の不整合 | 中 | ファイル形式の標準化、バリデーション |
| 並列実行時の競合 | 低 | 各subagentは独立したファイルに出力 |
| プロンプトインジェクション | 低 | プロンプトテンプレートの静的定義 |

### 2.3 リスク評価

**全体リスク: 低**

理由：
1. 変更対象は内部ツール（外部公開なし）
2. ファイル編集制限は既存機構で担保
3. 機密データの取り扱いなし

## 3. セキュリティ要件

### SR-001: フェーズ制限の継続
- 既存のhooksによるファイル編集制限を維持
- subagentも同じ制限を受ける

### SR-002: 成果物の検証
- 各フェーズの成果物が存在することを確認してから次フェーズへ
- 既存のartifact checkフックを継続使用

## 4. 結論

本機能はセキュリティリスクが低く、既存のセキュリティ機構で十分に対応可能。
