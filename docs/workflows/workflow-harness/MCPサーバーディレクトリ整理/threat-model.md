# MCPサーバーディレクトリ整理 - 脅威モデル

## 概要

ディレクトリ移動とファイル削除のみのリファクタリングタスク。セキュリティリスクは最小限。

## 脅威分析

### TH-1: 設定ファイルのパス不整合

| 項目 | 内容 |
|------|------|
| 脅威 | 移動後にMCPサーバーが起動できなくなる |
| 影響 | 低（開発環境のみ） |
| 対策 | 移動後に設定ファイルのパスを確認・更新 |

### TH-2: 誤ったファイル削除

| 項目 | 内容 |
|------|------|
| 脅威 | 必要なファイルを誤って削除 |
| 影響 | 中 |
| 対策 | 削除前にファイル内容を確認、Gitで復元可能 |

## リスク評価

| リスク | 発生確率 | 影響度 | 総合 |
|--------|---------|--------|------|
| パス不整合 | 低 | 低 | 低 |
| 誤削除 | 低 | 中 | 低 |

## 結論

本タスクはセキュリティリスクが低く、Gitによるバージョン管理下で安全に実行可能。
