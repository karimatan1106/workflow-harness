## MCPサーバー再起動の必須性

CLAUDE.md のルール 22「コアモジュール変更後は MCPサーバーを再起動してから次フェーズに進むこと」に従い、MCP サーバーの再起動が必須です。

### 変更対象のコアモジュール

本タスクで変更したファイル:

- **`workflow-plugin/mcp-server/src/phases/definitions.ts`**: サブエージェントテンプレートのサマリーセクション最小行数要件を仕様書に合わせて統一。また Mermaid フローチャート記法の制約ガイドラインを追加。

このファイルは MCP サーバーの起動時に require() またはimport でメモリにキャッシュされます。修正をユーザーが使用する環境に反映するには、MCPサーバープロセスの再起動が必須です。

### Node.js モジュールキャッシング機構

MCP サーバー起動時、Node.js はロードされたモジュールをグローバルキャッシュに保存します。dist/*.js ファイルをディスク上で変更しても、実行中の MCPサーバープロセスはキャッシュ版を使用し続けます。

キャッシュ版には以下の不具合が生じる可能性があります:

- サブエージェントテンプレートの古い最小行数要件でバリデーション失敗が継続する
- flowchart 記法ガイドラインが反映されず、新しい制約がバリデーターで認識されない
- parallel_verification フェーズで複数のリトライが発生し続ける

これらの問題を回避するため、必ず MCPサーバーを再起動して新しいモジュールをメモリにロードする必要があります。

## MCPサーバー再起動の実施者

**重要**: MCPサーバーの再起動はローカル開発環境の操作であり、このサブエージェントからは実行不可能です。

MCPサーバー再起動は以下のいずれかの方法でユーザーが実施します:

- Claude Desktop GUI の「開発」→「サーバー管理」からサーバー再起動ボタンをクリック
- コマンドラインから MCPサーバープロセスを終了し、Claude Desktop を再起動
- システムレベルでプロセスをkill（Unix/Linux/macOS）またはタスクマネージャーで終了（Windows）

本デプロイフェーズでは、再起動が必要であることをユーザーに伝え、ユーザーが手動で再起動を実施する流れです。

## トラブルシューティング

### MCPサーバーが起動しない場合

1. コンソールエラーメッセージを確認
2. workflow-plugin/mcp-server/dist/ ディレクトリが存在することを確認
3. npm run build を再実行してビルドエラーを確認
4. Node.js バージョン確認（LTS 推奨）

### バリデーション失敗が継続する場合

MCPサーバーのモジュールキャッシュが古い可能性があります:

1. MCPサーバープロセスが完全に終了したことを確認（プロセスマネージャーで検索）
2. Claude Desktop を完全に閉じて再起動
3. 新しいワークフローを開始してテンプレートが更新されたことを確認

### サマリーセクション行数不足エラーが再発する場合

definitions.ts の変更が正常に反映されていない可能性があります:

1. git log で最新コミット ee094cc が適用されていることを確認
2. npm run build が成功し、dist/phases/definitions.js が更新されたことを確認
3. MCPサーバー再起動時刻がビルド時刻より後であることを確認
