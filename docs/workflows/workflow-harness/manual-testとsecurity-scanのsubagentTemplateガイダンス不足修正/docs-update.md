## サマリー

本タスク（FR-11・FR-12）では、parallel_verification フェーズの manual_test および security_scan subagentTemplate に対して、成果物品質要件を満たすための具体的なガイダンスを追記しました。
FR-11 は manual_test の総合評価セクション記述指針、FR-12 は security_scan の行数確保計算ロジックを新たに definitions.ts へ実装し、両フェーズのテンプレートに埋め込むことで、subagent が Orchestrator からガイダンスを自動受け取りできる体系を整備しました。

## 永続ドキュメント更新の判断

### CLAUDE.md への更新が不要な理由

CLAUDE.md の「manual_test」および「security_scan」セクション（線548-580）は、以下の理由により更新不要と判断しました。

**設計方針の一貫性**: CLAUDE.md はエンタープライズレベルのプロダクト開発ガイドとして、ユーザー・開発者向けの概要情報を提供する位置付けです。
一方、subagent が従うべき詳細なプロンプトガイダンス（実質行数確保方法・重複行回避パターン・値観点別記述指針）は definitions.ts の subagentTemplate に埋め込まれ、Orchestrator を経由して subagent へ直接渡されます。
この階層分離により、CLAUDE.md は安定的な概要記述、definitions.ts は詳細かつ動的に更新可能な仕様書という役割分担が実現されています。

**保守性と冗長性の懸念**: CLAUDE.md に subagentTemplate と同一の詳細ガイダンスを記載すると、両ドキュメント間のドリフト（乖離）が発生し、更新時に両方を修正する保守負荷が増加します。
既に実装済みの FR-1～FR-4（2026-02-23）も同様に CLAUDE.md には記載されておらず、definitions.ts のテンプレートのみに集約されています。
この先例に従い、FR-11・FR-12 の詳細ガイダンスも CLAUDE.md への記載を見送ります。

**ユーザー向け情報の充足性**: CLAUDE.md の現在の記述「必須セクション」「編集可能ファイル」「目的」は、ユーザーが manual_test・security_scan フェーズの概要を理解するために十分です。
詳細なテンプレートガイダンスはユーザーが直接参照する情報ではなく、subagent が Orchestrator を経由して自動受け取りする内部仕様であるため、CLAUDE.md に記載する必要がありません。

**検証**: 2026-02-23 の FR-1～FR-4 実装後に CLAUDE.md が更新されていないことが、このドキュメント戦略の正当性を示しています。

## FR-11・FR-12 実装内容の確認

### FR-11: manual_test 総合評価セクション記述指針

実装位置: workflow-plugin/mcp-server/src/phases/definitions.ts, manual_test.subagentTemplate, 線906 付近

**追加ガイダンス**: 「## ★ 総合評価セクションの記述指針（FR-11）」

総合評価セクションに必須の5観点を明示：
- 全テストシナリオの合否サマリー（実施件数・合格件数・不合格件数を数値で評価）
- 検出された問題の有無・件数・深刻度（不具合が存在しない場合は「不具合なし」と判断した根拠を記述）
- 未実施シナリオがある場合の理由と代替措置
- 次フェーズ（security_scan等）への引き継ぎ事項
- 全体的な品質評価（合格・条件付き合格・不合格のいずれかと根拠）

**効果**: subagent が総合評価セクションで実質行不足（5行未満）となるリスクを低減し、バリデーション失敗の頻度を削減

### FR-12: security_scan 行数確保計算ロジック

実装位置: workflow-plugin/mcp-server/src/phases/definitions.ts, security_scan.subagentTemplate, 線918 付近

**追加ガイダンス**: 「## ★ 行数確保の記述指針（FR-12）」

minLines（20行）達成のための具体的な計算ロジックを明示：
- 必須3セクション（脆弱性スキャン結果・検出された問題・サマリー）に各5行 = 合計15行
- 残り5行以上を追加セクション（総合評価等）で確保
- 合計20行以上に達することでminLines要件を充足

**効果**: security_scan の成果物がセクション密度不足（各セクション5行未満）でバリデーション失敗となる問題を、より明確で計算可能なガイダンスにより回避
