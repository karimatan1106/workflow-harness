## サマリー

本脅威モデルは、`workflow-plugin/mcp-server/src/phases/definitions.ts` の `manual_test` フェーズおよび `security_scan` フェーズの subagentTemplate に対するガイダンス追加（FR-11・FR-12）を対象としている。

- 目的: テンプレート文字列修正に際して発生しうるセキュリティ上のリスクおよび品質上のリスクを洗い出し、実装時に対処すべき事項を明確化する。
- 主要な決定事項: 変更対象はテンプレート文字列という読み取り専用の静的定数であり、実行時の動的な入力処理や認証制御には関与しない。そのため外部攻撃ベクターは非常に限定的であるが、内部品質リスク（バリデーション失敗・テストリグレッション）は実際に発生しうる。
- 特定されたリスク数: 内部品質リスク4件（禁止語混入・テストリグレッション・行数変化・再起動省略）および間接リスク1件（重複行エラー）の合計5件を洗い出した。
- 対応優先度の判定: 禁止語混入（シナリオ1）とMCPサーバー再起動省略（シナリオ4）は深刻度「高」かつ発生確率「中」であるため「最優先」対応として位置づけた。
- 次フェーズで必要な情報: リスク評価の結果として、実装時には「禁止語12語の混入防止」「既存テスト912件のリグレッション確認」「テンプレート行数の妥当性確認」の3点を必ず実施すること。

## 脅威シナリオ

### シナリオ1: 禁止語の混入によるバリデーション失敗（内部品質リスク）

ガイダンステキストを追記する際、artifact-validatorが検出する12語（英語グループ4語・日本語グループ8語）をコードフェンス外に記述するリスクがある。
このリスクは、ガイダンスの説明文として「後続フェーズの対処事項が確定していない場合は……」のような表現を使ったときに潜在的に発生する。
禁止語を含む語句が1つでもテンプレート文字列に混入すると、そのテンプレートを使用して生成された成果物がバリデーション失敗となり、subagentがエラーループに入る。
影響範囲は `manual_test` と `security_scan` の2フェーズに限定されるが、parallel_verification全体が停止する可能性がある。
過去の実績として、FR-1〜FR-10の修正時にも禁止語混入によるバリデーション失敗が発生したことが記録されている。

### シナリオ2: 既存テストスイートへのリグレッション（内部品質リスク）

definitions.tsは912件のテストが存在するテスト済みモジュールであり、テンプレート文字列を変更するとテストケースの期待値と実際の文字列内容が乖離する可能性がある。
FR-11の追加内容（`## ★ 総合評価セクションの記述指針（FR-11）` ブロック）がテスト対象の文字列に含まれることをテストで検証している場合、追加前後でテスト結果が変わる可能性がある。
既存のFR-6〜FR-10テストは `toContain()` 形式で記述されているため、追加前は新規キーワードに関するテストが失敗する（TDD Redフェーズ相当）。
逆に、既存テストが参照しているテンプレートの構造を崩す変更を加えた場合、従来通過していたテストが失敗する（リグレッション）。

### シナリオ3: テンプレート行数・密度の変化によるサブエージェント誤動作（内部品質リスク）

subagentTemplateはminLines・minSectionLinesの要件を満たした成果物を生成するためのガイダンスを提供する。
追加するガイダンスブロックの行数が過剰であった場合、テンプレートそのものがバリデーターの行数上限に近づく可能性がある（現状では理論的なリスクに留まる）。
追加するガイダンスが不十分であった場合、subagentがガイダンスを理解できず、引き続きバリデーション失敗を繰り返す問題が継続する。
ガイダンスの記述スタイルが既存ブロック（FR-9等）と大きく異なる場合、subagentが矛盾する指示として解釈し混乱するリスクがある。

### シナリオ4: MCPサーバー再起動省略による変更未反映（運用リスク）

Node.jsのモジュールキャッシュ機構により、definitions.tsのビルド後に再起動を省略した場合、変更がメモリ上に反映されない。
再起動を省略した状態でworkflow_nextを呼び出しても、変更前のテンプレートが使われ続け、FR-11・FR-12の効果が発揮されない。
この状態は無声的な失敗であり、ガイダンスが正しく追加されたにもかかわらず効果がないように見えるため、原因の特定が遅れるリスクがある。
CLAUDE.mdおよびMEMORY.mdに再起動必須の手順が明記されているにもかかわらず、過去に省略による障害が複数回発生している実績がある。

### シナリオ5: 成果物の重複行エラーによるサブエージェントリトライ増加（品質リスク）

FR-11で追加する5観点ガイダンスの記述例として、同一パターンの例示行（NG例・OK例）を列挙する際に、3行以上が完全一致する可能性がある。
具体的には「OK: 「- 観点1: xxxについて記述する」」「OK: 「- 観点2: yyyについて記述する」」といったOK例が形式的に同一の場合、artifact-validatorの重複行検出が作動する。
ただし、本リスクはテンプレートの内容が成果物ではなく定義ファイルに記述されるため、definitions.ts本体への直接影響はない。
subagentがガイダンスを参考にして成果物を作成する際に、ガイダンスのOK例をそのままコピーする行動パターンが発生した場合に間接的なリスクとなる。

## リスク評価

| リスク | 深刻度 | 発生確率 | 対策優先度 |
|--------|--------|----------|-----------|
| シナリオ1（禁止語混入） | 高 | 中 | 最優先 |
| シナリオ2（テストリグレッション） | 中 | 低 | 優先 |
| シナリオ3（行数・密度変化） | 中 | 低 | 優先 |
| シナリオ4（再起動省略） | 高 | 中 | 最優先 |
| シナリオ5（重複行エラー間接影響） | 低 | 低 | 低優先 |

各リスクの評価根拠を以下に記述する。

シナリオ1の深刻度が「高」である理由は、1語の混入で成果物全体がバリデーション失敗となるためであり、エラーが即座に顕在化する点で検出は容易だが、フィードバックループが形成されると連鎖的にリトライが増加する。

シナリオ2の発生確率が「低」である理由は、FR-11・FR-12のテストケースはTDD Redフェーズとして追加するため、実装前に意図的に失敗させる設計であり、既存テストへの波及が構造的に限定されているためである。

シナリオ4の深刻度が「高」である理由は、再起動省略は変更効果がゼロになる無声的失敗であり、問題の発見が遅れやすいためである。

シナリオ5の発生確率が「低」である理由は、subagentはガイダンスを参考にして独自の成果物を生成する傾向があり、ガイダンスのOK例をそのまま複製するパターンはあまり見られないためである。

## セキュリティ要件

FR-11・FR-12の実装にあたり、以下のセキュリティ要件を必ず満足すること。

禁止語チェック（シナリオ1対策）として、追加するガイダンステキスト全文に対して英語グループおよび日本語グループのいずれの語句も含まれていないことを、テキストエディタの検索機能またはgrepコマンドで事前確認すること。
確認対象は追加するテンプレート文字列の全行であり、コメント行も含む。
禁止語が含まれる場合は、CLAUDE.mdの「安全な代替表現パターン」セクションに従って言い換えを行うこと。

ビルドおよび再起動手順（シナリオ4対策）として、definitions.ts修正後は必ず以下の手順を順守すること。
第1に `cd workflow-plugin/mcp-server && npm run build` を実行してTypeScriptをトランスパイルすること。
第2に `dist/*.js` ファイルの更新日時が現在時刻であることを確認すること。
第3にMCPサーバープロセスを終了して再起動すること。
第4に `workflow_status` を呼び出してフェーズが正しく認識されることを確認すること。

テストリグレッション防止（シナリオ2対策）として、FR-11・FR-12のテストケースをtest_implフェーズで先に追加（TDD Red）し、implementation後にテストスイート全体を実行して既存の912件がすべてパスすることを確認すること。
テスト実行コマンドは `cd workflow-plugin/mcp-server && npm test` であり、実行後のテスト件数が実装前より減少していないことを確認すること。

ガイダンスパターン一貫性（シナリオ3対策）として、追加するFR-11ガイダンスブロックの構造は既存のFR-9ブロック（performance_testの `## ★ 総合評価セクションの記述指針（FR-9）`）と同一パターンを採用すること。
見出し形式は `## ★ 総合評価セクションの記述指針（FR-11）` とし、5観点の列挙構造・文体・行構成をFR-9と揃えること。
FR-12の計算ロジックは既存の「行数カウント仕様と転記防止」セクションへの追記として実施し、新規セクションを設けずに既存ガイダンスの文脈を維持すること。
