## パフォーマンス評価

SEC-ENV-1のcheckSecurityEnvVar内でループごとにnew RegExpを実行している点がある。

ただし呼び出し頻度を考慮すると実用上の影響は軽微である。
BUG-2の密度計算は一度の走査でカウントするO(n)の効率的な実装である。
SEC-TIME-1のBuffer長チェックで早期リターンによりtimingSafeEqual呼び出しを最小化する。
SEC-LOG-1のマスキング処理は対象文字列が100文字以下に制限されるため影響は無視できる。
