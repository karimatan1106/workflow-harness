## ステートマシン図の技術的特徴

### 1. 状態遷移の明確性

```mermaid
State A --条件--> State B
```

全てのエッジに条件やアクションを明記し、不確実な遷移がないことを保証

### 2. 並列フェーズ対応

```
parallel_design フェーズ
├─ state_machine (アクティブ)
├─ flowchart
└─ ui_design
```

複数サブフェーズが並行実行される場合の処理フロー

### 3. 安全側への倒し方

```
不確実な場合は許可（Allowed）へ遷移
例: 不明なフェーズ、ファイルパス抽出失敗、JSON パースエラー等
```

### 4. スコープ検証の階層構造

```
implementation フェーズのみ実行
├─ docs/ 配下 → 常に許可
├─ src/ 配下 → affectedFiles/affectedDirs チェック
└─ その他 → 許可
```

## 主要な制御メカニズム

### メカニズム1: Bashコマンド検証

```
危険パターン (kill, shutdown, rm -rf等) を正規表現でマッチング
```

**検出パターン**:
- PowerShell系: stop-process, remove-item
- Windows系: taskkill, wmic
- Unix系: kill -9, killall, pkill
- システム終了: shutdown, reboot, halt
- ファイル破壊: rm -rf, format c:
- バイパス対策: bash -c 'kill...', powershell -command 'stop-process'

### メカニズム2: フェーズ別編集制限

```
現在のフェーズ + ファイルタイプ → allowed/blocked 配列で判定
```

**フェーズ別ルール例**:
- research: spec のみ許可 (code, test 禁止)
- test_impl: test のみ許可 (code 禁止) ← TDD Red
- implementation: code のみ許可 (test 禁止) ← TDD Green
- refactoring: 全て許可 ← TDD Refactor
- 読み取り専用フェーズ: 全て禁止 (build_check, testing等)

### メカニズム3: ファイルタイプ判定（優先度順）

```
1. テストファイル (.test., .spec.)
2. 図式ファイル (.mmd)
3. 仕様書 (.md)
4. ソースコード (.ts, .tsx等)
5. 設定ファイル (package.json等)
6. 環境変数ファイル (.env)
7. その他
```

### メカニズム4: 並列フェーズ処理

```
並列フェーズ (parallel_design) の場合:
  1. アクティブサブフェーズを特定
  2. サブフェーズ不明の場合は合算ルール適用
```

### メカニズム5: スコープ検証（implementationフェーズ）

```
ファイルが affectedFiles または affectedDirs に含まれるか確認
含まれない場合はブロック
```

## ログ出力例

### 許可ログ

```json
{
  "timestamp": "2026-02-07T12:00:00.000Z",
  "allowed": true,
  "phase": "implementation",
  "filePath": "src/main.ts",
  "fileType": "code"
}
```

### ブロックログ

```json
{
  "timestamp": "2026-02-07T12:01:00.000Z",
  "blocked": true,
  "phase": "research",
  "filePath": "src/main.ts",
  "fileType": "code",
  "reason": "research フェーズでは code は禁止"
}
```

## ブロックメッセージの内容

ユーザーがブロックされた場合、以下を表示:

1. **フェーズ情報**: フェーズ名（日本語表記）
2. **ファイル情報**: ファイルパス、ファイルタイプ
3. **理由**: なぜブロックされたか
4. **TDD情報**: 該当する場合（Red/Green/Refactor）
5. **許可ファイル一覧**: 編集可能なファイルタイプ
6. **次のステップ**: どのフェーズに進めば編集可能か
7. **スキップ方法**: 緊急時のバイパス方法
