## テスト実行手順

### 事前準備

1. APIサーバーが起動していることを確認
2. データベースが初期化されていることを確認
3. テストユーザーが作成されていることを確認
4. ポート 8000 が利用可能であることを確認

### テスト実行

```bash
# curl を使用したテスト例

# 1. タスク一覧を取得
curl -X GET http://localhost:8000/api/workflow/status

# 2. 新規タスクを開始
curl -X POST http://localhost:8000/api/workflow/start   -H "Content-Type: application/json"   -d '{"taskName": "テストタスク"}'

# 3. タスク詳細を取得
curl -X GET http://localhost:8000/api/workflow/status/TASK_ID

# 4. フェーズを進行
curl -X POST http://localhost:8000/api/workflow/TASK_ID/next   -H "Content-Type: application/json"   -d '{}'
```

各テストケース実行後、以下の項目を記録してください:

## 品質基準

### 合格基準

- 全テストケースの成功率が 95% 以上
- 重大なセキュリティ脆弱性がない
- パフォーマンステストが全てパス（1秒以内）

## リスク評価

### 高リスク領域

1. **パストトラバーサル攻撃**: ファイルアクセス制御が不十分な場合
2. **エラーハンドリング**: 不適切なエラーメッセージがセキュリティ情報を露出
3. **並列フェーズ**: サブフェーズ管理の複雑性

### 対策

1. セキュリティテストの入念な実施
2. エラーメッセージの事前確認
3. 複数環境でのテスト実施
