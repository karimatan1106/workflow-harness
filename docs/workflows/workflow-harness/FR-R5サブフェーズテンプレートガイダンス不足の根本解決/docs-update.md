## サマリー

- 目的: FR-R5タスクの実装完了を確認し、外部ドキュメント更新の必要性を検証する
- 主要な決定事項: 本タスクの変更は内部実装のみであり、ユーザー向け外部ドキュメントの更新は不要であることを確認した
- 次フェーズで必要な情報: MCPサーバーの再ビルド・再起動により、新しいガイダンスパターンがsubagentに提供される状態

## タスク概要と変更の性質

FR-R5タスク（FR-R5サブフェーズテンプレートガイダンス不足の根本解決）は、ワークフロープラグインの並列検証フェーズ（parallel_verification）に含まれる4つのサブフェーズ（security_scan・code_review・e2e_test・manual_test）のsubagentTemplateプロパティにガイダンス文字列を追加するものである。

変更内容の性質:
- **変更ファイル**: workflow-plugin/mcp-server/src/phases/definitions.ts（1ファイルのみ）
- **変更範囲**: 4つのサブフェーズのsubagentTemplateプロパティへのガイダンス文字列追加
- **変更の構造**: テンプレート文字列の拡充のみ（設定値やロジック変更なし）
- **外部への影響**: なし

## CLAUDE.mdへの影響確認

CLAUDE.mdには以下の内容が記載されており、定期的に参照される重要なドキュメントである：

確認された内容:
1. **フェーズ別subagent設定表** - 各フェーズの入出力ファイル定義、使用モデル、重要度
2. **必須セクション定義** - manual_test、security_scan、performance_test、e2e_testの必須セクション記載
3. **Bashコマンド許可カテゴリ** - フェーズごとの許可コマンドカテゴリ定義
4. **成果物品質要件** - バリデーション要件（行数・密度・禁止パターン等）

確認結果:
- subagentTemplateという単語そのもの、またはテンプレート文字列の具体内容はCLAUDE.mdに記載されていない
- CLAUDE.mdに記載されているのは各フェーズ・サブフェーズの要件定義と入出力ファイル定義のみ
- requiredSections・minLines・minSectionLines等のバリデーション設定値も一切変更されていない
- したがって、CLAUDE.mdの更新は不要である

## 外部向けドキュメント対象の確認

docs/配下に配置されるドキュメント:
- docs/spec/ - プロダクト仕様（永続的）
- docs/architecture/ - システム設計・ADR
- docs/security/ - セキュリティ関連
- docs/testing/ - テスト計画・結果
- docs/operations/ - 運用関連
- docs/workflows/ - ワークフロー成果物（一時的）

確認内容:
- 上記ドキュメントはいずれもアプリケーションの機能・設計・運用に関するもの
- FR-R5タスクはワークフロープラグインの内部実装であり、アプリケーション仕様には無関係
- ガイダンス文字列の追加はsubagent起動時の情報提供改善であり、ユーザーに直接影響を与えない
- したがって、外部ドキュメント更新は不要である

## 成果物品質要件への準拠確認

本タスクで作成される成果物（subagentTemplateの修正内容）について:

準拠確認項目:
1. **テンプレート文字列の正確性**: performance_testで確立されたガイダンスパターンを踏襲し、セクション別必須項目列挙・NG/OK例を統一形式で記述する
2. **バリデーション設定値の変更なし**: requiredSections・minLines・minSectionLinesは変更しない（後方互換性確保）
3. **フェーズロジックへの影響なし**: フェーズ遷移ロジックには一切手を加えない
4. **既存フェーズへの影響なし**: research・requirements等の他フェーズのテンプレートには触れない

## MCPサーバー再ビルド・再起動の必要性

definitions.tsはMCPサーバーのコアモジュールである。変更を反映させるには以下の手順が必須:

1. npm run build - TypeScriptをトランスパイルし、dist/*.jsに変更を反映
2. MCPサーバープロセスの再起動 - モジュールキャッシュをクリア
3. workflow_statusで確認 - 現在のフェーズを確認し、同フェーズから作業再開

この手順により、新しいガイダンスパターンがsubagentに提供される。
