# 設計-実装整合性の自動検証機能 要件定義

## 機能要件

### FR-1: 設計項目の自動抽出

| ID | 要件 | 優先度 |
|----|------|--------|
| FR-1.1 | spec.mdからクラス名・メソッド名・ファイルパスを抽出できる | 必須 |
| FR-1.2 | state-machine.mmdから全状態・全遷移を抽出できる | 必須 |
| FR-1.3 | flowchart.mmdから全プロセス・全決定点を抽出できる | 必須 |
| FR-1.4 | requirements.mdから全FR/NFR/AC項目を抽出できる | 必須 |
| FR-1.5 | ui-design.mdから全UI要素・コンポーネントを抽出できる | 必須 |
| FR-1.6 | test-design.mdから全テストケースIDを抽出できる | 任意 |

### FR-2: 実装状況の照合

| ID | 要件 | 優先度 |
|----|------|--------|
| FR-2.1 | 設計で指定されたファイルパスにファイルが存在するか確認 | 必須 |
| FR-2.2 | 設計で指定されたクラス・メソッドがコード内に存在するか確認 | 必須 |
| FR-2.3 | 状態遷移図の全状態に対応するコードが存在するか確認 | 任意 |
| FR-2.4 | フローチャートの全プロセスに対応するコードが存在するか確認 | 任意 |

### FR-3: フェーズ遷移のブロック

| ID | 要件 | 優先度 |
|----|------|--------|
| FR-3.1 | implementation開始前に設計完了を検証し、未完了ならブロック | 必須 |
| FR-3.2 | code_review開始時に設計-実装整合性を検証 | 必須 |
| FR-3.3 | 未実装項目の詳細リストをエラーメッセージで返す | 必須 |
| FR-3.4 | ブロック時に「何を実装すべきか」を明示する | 必須 |

### FR-4: 設定オプション

| ID | 要件 | 優先度 |
|----|------|--------|
| FR-4.1 | 厳格モード（未完了でブロック）と警告モード（警告のみ）を選択可能 | 任意 |
| FR-4.2 | 環境変数 `VALIDATE_DESIGN_STRICT` で切り替え | 任意 |
| FR-4.3 | 環境変数 `SKIP_DESIGN_VALIDATION` で無効化可能 | 任意 |

## 非機能要件

### NFR-1: パフォーマンス

| ID | 要件 | 目標値 |
|----|------|--------|
| NFR-1.1 | 検証処理は3秒以内に完了すること | 3秒 |
| NFR-1.2 | 大規模プロジェクト（100ファイル以上）でも動作すること | 100ファイル |

### NFR-2: 信頼性

| ID | 要件 | 目標値 |
|----|------|--------|
| NFR-2.1 | パース失敗時はエラーを許容し、処理継続すること | - |
| NFR-2.2 | 検証失敗時の理由を明確に表示すること | - |

### NFR-3: 保守性

| ID | 要件 | 目標値 |
|----|------|--------|
| NFR-3.1 | 新しい設計書フォーマットを追加しやすい構造にすること | - |
| NFR-3.2 | 抽出ルールを設定ファイルで変更可能にすること | - |

## 受け入れ基準

### AC-1: 設計項目抽出

- [ ] spec.mdからクラス・メソッド・ファイルパスを正しく抽出できる
- [ ] state-machine.mmdから全状態と遷移を抽出できる
- [ ] flowchart.mmdから全プロセスと決定点を抽出できる
- [ ] requirements.mdから全FR/NFR/AC IDを抽出できる

### AC-2: フェーズ遷移ブロック

- [ ] implementation開始時に設計未完了ならブロックされる
- [ ] code_review時に未実装項目があれば報告される
- [ ] ブロック時のエラーメッセージに未完了項目リストが含まれる

### AC-3: 既存ワークフローとの統合

- [ ] workflow_nextコマンドに検証ロジックが統合される
- [ ] 既存のフェーズ遷移ロジックを壊さない
- [ ] 既存の並列フェーズ完了チェックと共存する

### AC-4: 設定オプション

- [ ] 環境変数で検証の厳格度を切り替えられる
- [ ] 検証を無効化するオプションがある
