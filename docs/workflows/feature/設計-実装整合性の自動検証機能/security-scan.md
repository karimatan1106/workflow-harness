## 2. 入力検証/ReDoS対策チェック

### 2.1 正規表現パターン分析

**ファイル**: `spec-parser.ts`

**パターン1: クラス抽出**
```
/class\s+(\w+)\s*[:{]/g
```
**評価**: ✅ PASS - シンプルで安全

**パターン2: メソッド抽出**
```
/(?:def\s+)?(\w+)\s*\([^)]*\)\s*(?::\s*\w+)?\s*[{=]/g
```
**評価**: ⚠️ MEDIUM - ReDoS対策必要

**問題点**:
- `[^)]*` は制限なしで任意の文字列にマッチ
- 長い括弧内テキストで、バックトラッキングが発生する可能性あり
- 例: `method((((((((((((((((((((((((((x` → 指数関数的な計算量

**推奨修正**: 長さ制限パターンに変更
```
/(?:def\s+)?(\w+)\s*\([^)]{0,100}\)\s*(?::\s*\w+)?\s*[{=]/g
```

**パターン3: ファイルパス抽出**
```
/`(src\/[^\s`]+)`/g
```
**評価**: ✅ PASS - シンプルで安全

### 2.2 入力長チェック

**問題**: 入力サイズの上限チェックがない

**リスク**: メモリ使用量の過剰化（正規表現スキャンで大容量ファイル）

**推奨修正**: 最大ファイルサイズ制限（例: 10MB）の実装

**推奨修正**: try-catchでのエラー処理追加
