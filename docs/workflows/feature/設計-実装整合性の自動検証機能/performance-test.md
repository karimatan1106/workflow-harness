## 正規表現パフォーマンス分析

### Spec Parser 正規表現

#### クラス抽出パターン
```regex
/class\s+(\w+)\s*[:{]/g
```
- **用途**: クラス定義の抽出
- **複雑度**: O(n) - 線形
- **パフォーマンス**: 高速（単純なメタ文字のみ）

#### メソッド抽出パターン
```regex
/(?:def\s+)?(\w+)\s*\([^)]*\)\s*(?::\s*\w+)?\s*[{=]/g
```
- **用途**: メソッド定義の抽出
- **複雑度**: O(n) - 線形
- **パフォーマンス**: 中程度（後読みはなし）

#### ファイルパス抽出パターン
```regex
/`(src\/[^\s`]+)`/g
/\*\*ファイル\*\*:\s*`([^`]+)`/g
```
- **用途**: ファイルパスの抽出
- **複雑度**: O(n) - 線形
- **パフォーマンス**: 高速

### 正規表現最適化状況
