# 設計-実装整合性の自動検証機能 UI設計

## 1. 概要

本機能はCLI/MCPツールのため、UIは主にコンソール出力とエラーメッセージで構成される。

## 2. 出力フォーマット

### 2.1 検証成功時

```
============================================================
 設計-実装整合性チェック: OK
============================================================

 完了率: 12/12 (100%)

 検証項目:
   ✓ spec.md: 3クラス, 8メソッド
   ✓ state-machine.mmd: 5状態, 7遷移
   ✓ flowchart.mmd: 6プロセス, 2決定点
   ✓ requirements.md: 4FR, 2NFR, 6AC

============================================================
```

### 2.2 検証失敗時（厳格モード）

```
============================================================
 設計-実装整合性チェック: 未完了項目があります
============================================================

 完了率: 8/12 (67%)

 未実装項目:
   - [class] DesignValidator (spec.md)
     期待パス: src/validation/design-validator.ts

   - [method] parseStateMachine (spec.md)
     期待パス: src/validation/design-validator.ts

   - [state] ParsingError (state-machine.mmd)
     対応コードが見つかりません

   - [process] ファイル存在チェック (flowchart.mmd)
     対応コードが見つかりません

 対応方法:
   1. 上記項目を実装してください
   2. または、設計書を修正して /workflow reset で戻る

 スキップ（緊急時のみ）:
   SKIP_DESIGN_VALIDATION=true を設定

============================================================
```

### 2.3 警告モード

```
============================================================
 設計-実装整合性チェック: 警告
============================================================

 完了率: 8/12 (67%)

 [警告] 以下の項目が未実装です:
   - [class] DesignValidator
   - [method] parseStateMachine
   - [state] ParsingError
   - [process] ファイル存在チェック

 警告モードのため、フェーズ遷移を続行します。

============================================================
```

### 2.4 スキップ時

```
[設計検証] SKIP_DESIGN_VALIDATION=true によりスキップ
```

## 3. エラーメッセージ設計

### 3.1 パースエラー

| エラー | メッセージ |
|--------|-----------|
| ファイル不存在 | `{file}が見つかりません。parallel_designフェーズで作成してください。` |
| フォーマットエラー | `{file}のパースに失敗しました: {詳細}` |
| タイムアウト | `検証処理がタイムアウトしました（3秒）。ファイルサイズを確認してください。` |

### 3.2 検証エラー

| エラー | メッセージ |
|--------|-----------|
| クラス未実装 | `[class] {name} が {expectedPath} に見つかりません` |
| メソッド未実装 | `[method] {className}.{methodName} が実装されていません` |
| 状態未実装 | `[state] {stateName} に対応するコードが見つかりません` |
| プロセス未実装 | `[process] {processName} に対応するコードが見つかりません` |

## 4. 進捗表示

### 4.1 検証中の進捗

```

[設計検証] state-machine.mmd をパース中...
[設計検証] flowchart.mmd をパース中...

[設計検証] 実装ファイルを照合中...
```

### 4.2 結果サマリー

```

```

## 5. カラー設計（ターミナル対応）

| 要素 | 色 | ANSIコード |
|------|-----|-----------|
| 成功 | 緑 | `\x1b[32m` |
| 警告 | 黄 | `\x1b[33m` |
| エラー | 赤 | `\x1b[31m` |
| 情報 | 青 | `\x1b[34m` |
| リセット | - | `\x1b[0m` |

## 6. 国際化対応

メッセージは日本語で統一（CLAUDE.mdの規約に従う）。
将来的な英語対応のため、メッセージはテンプレート化する。

```typescript
const MESSAGES = {
  ja: {
    checkPassed: '設計-実装整合性チェック: OK',
    checkFailed: '設計-実装整合性チェック: 未完了項目があります',
    // ...
  }
};
```
