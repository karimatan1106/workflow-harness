## Threat Model: gitignore整備

### 脅威1: HMAC秘密鍵の履歴残存
- 影響: hmac-keys.json がGit履歴に残りクローン時に取得可能
- 緩和策: 本タスクのスコープ外として分離。別タスクでBFG等による履歴浄化を実施
- 残存リスク: 中（プライベートリポジトリのため外部露出リスクは低い）

### 脅威2: git rm --cached の誤操作
- 影響: 意図しないファイルが追跡対象から除去される
- 緩和策: ディレクトリ単位で個別実行し各操作後に git status で確認
- 残存リスク: 低（対象パスを明示的に指定）

### 脅威3: サブモジュールコミット忘れ
- 影響: 親リポジトリのサブモジュール参照が古いまま
- 緩和策: 実装手順でサブモジュール内コミットを親コミットより先に定義
- 残存リスク: 低（手順で順序を強制）
