# ADR-002: フェーズ = コンテキスト圧縮

Status: accepted
Date: 2026-03-01

## Context
LLMのコンテキストウィンドウは有限。大規模タスクでは情報が溢れ、品質が劣化する。

## Decision
30フェーズ構造を「コンテキスト圧縮メカニズム」として設計する。
各フェーズの成果物が次フェーズへの完全な引き継ぎとなる。

## Rationale
- 各フェーズは狭いスコープに集中 → LLMの注意が分散しない
- 成果物(TOON)が圧縮された文脈 → 前フェーズの全情報を要約して渡す
- コンテキストウィンドウを使い切る前にフェーズ境界でリセット可能
- 10Mトークンセッションでも安定動作するための設計

## Consequences
- フェーズ間の成果物が「完全な引き継ぎ」になっていないとロスが発生する
- フェーズ粒度が細かすぎるとオーバーヘッド、粗すぎるとコンテキスト溢れ
- small/medium/largeのサイジングでフェーズ数を調整して対応
