# ADR-004: ドキュメント3層分離（Why / What / How）

Status: accepted
Date: 2026-03-20

## Context
CLAUDE.mdにWhy/What/Howが混在していた。記事「仕様駆動開発(SDD)から意図駆動開発(IDD)へ」の知見を受け、ドキュメントを3層に分離した。

## Decision
ドキュメントを以下の3層に分離し、各層の責務を固定する。

### Why層: docs/adr/ADR-NNN.md
- 判断の記録。「なぜそのルールにしたか」「なぜそうしなかったか」
- イミュータブル（追記のみ）。変更時は新ADRで撤回・更新
- 仕様書との違い: 仕様書は「現在の正解」→コードと乖離。ADRは「その時点の判断」→乖離しない

### What層: CLAUDE.md
- ルール・制約の一覧。「何を守るか」のみ
- Howを書かない。手順・実装詳細はスキルファイルまたはコードに委ねる
- ADRへのポインタ（→ ADR-NNN）でWhyを参照

### How層: スキルファイル + コード
- スキルファイル(.claude/skills/): LLMへの手順指示（コードで強制できないHow）
- hooks/DoD/registry: コードで強制するHow（ドキュメント不要）

## 判断基準: 新しい内容をどこに書くか

| 問い | 答え | 書く場所 |
|------|------|---------|
| なぜこのルールにしたか？ | 判断の経緯・却下した代替案 | ADR（Why層） |
| 何を守るべきか？ | ルール・制約・禁止事項 | CLAUDE.md（What層） |
| どうやって実行するか？ | 手順・フロー・設定 | スキルファイル or コード（How層） |

## 追加の判断基準

- コードで強制できるか？ → Yes: hookやDoDに実装。ドキュメント不要
- コードで強制できないか？ → Yes: スキルファイルにHow記述
- CLAUDE.mdに書こうとしている内容にHowが含まれているか？ → Yes: スキルファイルに移動
- 「なぜ」が書かれているか？ → Yes: ADRに移動、CLAUDE.mdにはポインタのみ

## Rationale
IDD（Intent Driven Development）の原則:
- Howをドキュメントに書くから二重管理になる
- Howはコード（またはLLM指示書）に任せる
- Whyは仕様書ではなくADR（イミュータブルな判断記録）として残す
- 「仕様ではなく意図。管理ではなく記録。同期ではなく追記。」

## Consequences
- CLAUDE.md修正時はWhat以外が混入していないか確認する
- 新ルール追加時は必ずADRも作成する（Whyなきルールは追加しない）
- スキルファイル修正時はCLAUDE.mdとの重複がないか確認する
